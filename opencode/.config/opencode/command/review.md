description: Review of the code changes
---

Review the current git diff and provide:

1. **Summary**: What changed? (affected modules/layers, functional vs technical changes, overall intent)

2. **Critical Issues** (if any):

- Bugs, security vulnerabilities, or breaking changes
- Logic errors or edge cases not handled

3. **Architecture & Design**:

- Changes that don't align with the codebase patterns
- Better approaches or refactoring opportunities
- Performance/algorithmic improvements

4. **Code Quality**:

- Clarity/readability improvements
- Type safety or validation gaps
- Error handling issues

5. **Housekeeping**:

- TODOs, commented code, or debug statements
- Missing tests or documentation
- Anything that seems out of place

6. **Documentation**:

- Point out updates (if any) that should be made to this projects docs (README, CONTRIBUTING...)

7. **Commit Message**: Suggest a concise commit message following this project's conventions

Focus on the top 3-5 most impactful suggestions. Include code examples for non-obvious recommendations.
